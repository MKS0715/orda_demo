# 🏔️ 오르다 (ORDA)

**흥미가 오르다, 체력이 오르다, 건강이 오르다**

초등학생이 자신의 체력 데이터를 스스로 기록하고, 변화를 읽고, 목표를 정하고, 함께 실천하는 데이터 기반 자기주도 건강 체력 관리 웹앱입니다.

> **대상 학년**: 초등학교 5~6학년 (3~6학년 적용 가능)
> **비용**: 전부 무료. 구독료나 예산이 들지 않습니다.

## 💡 오르다는 어떤 프로그램인가

### ORDA 4단계

| 단계 | 이름 | 학생이 하는 일 |
|---|---|---|
| **O** | Observe · 나를 재다 | 4종목 체력을 직접 측정하고 기록을 입력한다 |
| **R** | Read · 데이터를 읽다 | 누적된 그래프를 보며 자신의 변화를 해석한다 |
| **D** | Decide · 목표를 정하다 | 부족한 체력 요소를 스스로 판단하고 목표를 세운다 |
| **A** | Act · 함께 실천하다 | 모둠과 함께 운동하고 챌린지 기록을 쌓는다 |

AI는 판단을 대신하지 않습니다. 학생이 자신의 데이터를 읽고 결정하도록 돕는 보조 도구로만 쓰입니다.

### 측정 4종목

| 체력 요소 | 종목 | 단위 |
|---|---|---|
| 🏃 심폐지구력 | 3분 왕복달리기 (15m 구간) | 회 |
| ⚡ 순발력 | 사이드스텝 | 회 / 20초 |
| 💪 근지구력 | 플랭크 | 초 (최대 180) |
| 🧘 유연성 | 윗몸 앞으로 굽히기 | cm |

주 1회씩 총 13주간 측정합니다. 1주차에 기준 기록을 재고, 2~12주차에 주 1회 측정과 챌린지를 병행하며, 13주차에 최종 측정과 자기 성찰을 합니다. 측정 방법과 안전 지도 사항은 [운영 가이드](https://mks0715.github.io/orda_demo/)에 있습니다.

## 📱 주요 기능

### 학생 화면

**📝 기록 입력**
4분할 입력 화면. 앱 안에 20초 타이머와 스톱워치가 내장되어 있어, 학생이 다른 앱으로 나갈 필요가 없습니다. 하루 1회만 저장되며, 측정하지 못한 종목은 비워두었다가 나중에 채울 수 있습니다.

**📊 성장 분석**
4종목 변화를 꺾은선 그래프로 보여줍니다. 2회 이상 기록이 쌓이면 생성형 AI가 이전 기록과 비교해 칭찬·격려·맞춤 운동을 제안합니다.

**👥 내 모둠 · 오르다 100 챌린지**
모둠 평균 체력을 보고 부족한 종목을 파악한 뒤, 일주일 동안 4종목 운동을 기록해 100점을 채웁니다. 모둠 성공 기준은 1인 평균 100점이라 모둠원 수가 달라도 공정합니다. 자세한 운영 방법은 [운영 가이드](https://mks0715.github.io/orda_demo/)를 참고하세요.

### 교사 화면

| 메뉴 | 기능 |
|---|---|
| 🔄 초기 세팅 | 클릭 한 번으로 시트 양식과 더미 학생 데이터 자동 생성 |
| 📝 기록 입력 | 결석·기기 문제 학생 대리 입력 (회차 자동 계산) |
| 👥 학생 관리 | 학생 추가·수정·삭제 (체력기록 동기화 포함) |
| 📊 전체 기록 | 학년·반별 전체 기록 열람 |
| 🏅 모둠 관리 | 모둠별 평균 체력, 챌린지 달성 현황, 누적 랭킹 |
| 🎯 챌린지 상세 | 학생 개인별 종목·주차별 참여 내역 확인 |

## 🛠 우리 반에 적용하기

준비물은 **구글 계정**, **GitHub 계정**(무료 가입), **학생용 태블릿 또는 스마트폰**입니다.

전체 흐름은 다음과 같습니다.

```
[1] 저장소 복제  →  [2] 구글 시트 + 서비스 계정  →  [3] AI 키(선택)
                            ↓
                  [4] Streamlit Cloud 배포  →  [5] 초기 세팅  →  [6] 모둠 등록
```

### 1단계 · 저장소 복제

이 페이지 우측 상단 **Fork** 버튼을 눌러 내 GitHub 계정으로 복사합니다.

- GitHub 계정이 없다면 먼저 [github.com](https://github.com) 에서 무료 가입하세요.
- Fork로 복사되는 것은 **코드뿐**입니다. 학생 데이터와 계정 키는 따라오지 않으므로, 다른 선생님의 학생 기록이 섞일 걱정은 없습니다.
- Fork 직후에는 아직 실행되는 앱이 아닙니다. 4단계 배포까지 마쳐야 나만의 주소가 생깁니다.

---

### 2단계 · 구글 시트와 서비스 계정 만들기

이 단계가 전체에서 가장 까다롭습니다. 순서를 그대로 따라가시면 됩니다.

**2-1. 스프레드시트 만들기**

구글 드라이브에서 새 스프레드시트를 만들고 이름을 정합니다. (예: `orda_myclass`)
이 **이름을 정확히 기억해 두세요.** 4단계에서 그대로 입력해야 합니다.

**2-2. 구글 클라우드 프로젝트 만들기**

1. [console.cloud.google.com](https://console.cloud.google.com) 접속 (구글 계정으로 로그인)
2. 상단의 프로젝트 선택 → **새 프로젝트** → 이름 입력 (예: `orda`) → 만들기

**2-3. API 두 개 사용 설정** ⚠️ **둘 다 켜야 합니다**

좌측 메뉴 **API 및 서비스 → 라이브러리** 에서 아래 두 개를 검색해 각각 **사용** 버튼을 누릅니다.

| API | 역할 |
|---|---|
| **Google Sheets API** | 시트의 값을 읽고 쓰기 |
| **Google Drive API** | 시트를 **이름으로 찾기** |

> Drive API를 켜지 않으면 앱은 배포에 성공하지만, 접속하는 순간 "스프레드시트를 찾을 수 없다"는 오류가 납니다. 가장 흔한 실패 원인입니다.

**2-4. 서비스 계정 만들기**

1. **API 및 서비스 → 사용자 인증 정보** → 상단 **사용자 인증 정보 만들기** → **서비스 계정**
2. 이름 입력 (예: `orda-bot`) → 만들고 계속하기 → 역할은 지정하지 않아도 됩니다 → 완료
3. 만들어진 서비스 계정을 클릭 → **키** 탭 → **키 추가 → 새 키 만들기 → JSON** → 만들기
4. JSON 파일이 자동으로 다운로드됩니다. **이 파일은 비밀번호와 같습니다. 절대 저장소에 올리지 마세요.**

**2-5. 시트에 서비스 계정 초대**

1. 다운로드한 JSON 파일을 메모장으로 열어 `client_email` 값을 복사합니다.
   (`orda-bot@orda-xxxxx.iam.gserviceaccount.com` 형태)
2. 2-1에서 만든 스프레드시트로 돌아가 **공유** 버튼을 누르고, 이 주소를 **편집자**로 추가합니다.

이 단계를 빠뜨리면 앱이 시트에 접근하지 못합니다.

### 3단계 · AI 피드백 키 발급 (선택)

[Google AI Studio](https://aistudio.google.com/apikey) 에서 **Gemini API 키**를 무료로 발급받습니다.

AI 피드백 없이 기록 입력과 성장 그래프만 써도 프로그램은 정상 작동합니다. 부담되면 이 단계를 건너뛰고 나중에 추가해도 됩니다.

### 4단계 · Streamlit Cloud 배포

1. [share.streamlit.io](https://share.streamlit.io) 에 **GitHub 계정으로 로그인**
2. **Create app** → **Deploy a public app from GitHub**
3. Repository: Fork한 내 저장소 선택 / Branch: `main` / Main file path: `app.py`
4. **Advanced settings** 를 펼쳐 **Python version: 3.11** 선택 ⚠️ **반드시**
5. 같은 화면의 **Secrets** 칸에 아래 내용을 채워 넣습니다.

```toml
spreadsheet_name = "orda_myclass"
admin_password = "교사용_관리자_비밀번호"
GEMINI_API_KEY = "발급받은_키"

[gcp_service_account]
type = "service_account"
project_id = "..."
private_key_id = "..."
private_key = "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
client_email = "...@....iam.gserviceaccount.com"
client_id = "..."
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "..."
```

작성 요령:

- `spreadsheet_name` : 2-1에서 정한 시트 이름과 **글자 하나까지 똑같이**
- `admin_password` : 교사 화면 진입 비밀번호. 직접 정하시면 됩니다.
- `[gcp_service_account]` 아래 : 2-4에서 받은 JSON 파일을 메모장으로 열어 값을 그대로 옮깁니다.
- `private_key` : 줄바꿈이 `\n` 문자로 되어 있는 그대로 한 줄에 붙여 넣습니다. 임의로 줄을 나누면 안 됩니다.

**Deploy** 를 누르고 2~3분 기다리면 나만의 주소가 만들어집니다.

> Python 버전을 지정하지 않으면 최신 버전이 잡혀 설치가 실패합니다. 배포가 안 될 때 가장 먼저 확인할 항목입니다.

---

### 5단계 · 초기 세팅

1. 배포된 앱에 접속 → **🔧 관리자 모드** → 4단계에서 정한 관리자 비밀번호 입력
2. **🔄 초기 세팅** → **🚀 초기 데이터 생성** 클릭
3. `학생명단`, `체력기록`, `챌린지기록` 세 개의 시트가 자동으로 만들어집니다
4. **👥 학생 관리**에서 더미 이름을 우리 반 학생 이름으로 바꿉니다

---

### 6단계 · 모둠 시트 만들기 (챌린지 사용 시)

모둠 탭은 자동 생성되지 않습니다. 구글 시트에 `모둠` 탭을 **직접 추가**하고 아래 형식으로 입력합니다.

| 학년 | 반 | 모둠번호 | 모둠이름 | 학생이름 |
|---|---|---|---|---|
| 5 | 1 | 1 | 오르미 | 김철수 |
| 5 | 1 | 1 | 오르미 | 이영희 |
| 5 | 1 | 2 | 튼튼이 | 박민수 |

- 탭 이름은 정확히 `모둠`, 헤더 순서도 그대로 지켜야 합니다.
- `학생이름`은 `학생명단` 시트의 이름과 정확히 일치해야 합니다.
- 이 시트가 없으면 모둠·챌린지 메뉴가 나타나지 않습니다.

## 🔧 문제가 생기면

실제 35명 규모의 교실 운영 중에 겪었던 문제와 해결 방법입니다. 이 저장소의 코드에는 아래 대응이 모두 반영되어 있습니다.

| 증상 | 원인 | 대응 |
|---|---|---|
| 비밀번호 앞자리 `0`이 사라짐 | 구글 시트의 `0203`을 Pandas가 숫자 `203`으로 읽음 | 문자열 변환 후 4자리 고정 |
| 학생이 스톱워치 앱을 켜면 로그아웃됨 | 모바일 기기가 메모리 정리를 하며 세션 초기화 | 앱 안에 타이머·스톱워치를 직접 내장 |
| 빈 회차가 계속 늘어남 | 저장 버튼 중복 클릭, 결측치 저장 | 하루 1회만 저장되도록 잠금 |
| 측정일이 하루 전으로 찍힘 | 서버 시간이 UTC 기준 | 한국 표준시(KST)로 보정 |
| 동시 접속 시 무한 로딩 | 구글 시트 API 읽기 제한 초과 (분당 60회) | 캐시 적용 및 재시도 로직 |

### 설정 단계에서 자주 나오는 오류

| 증상 | 확인할 것 |
|---|---|
| 배포 자체가 실패한다 | Streamlit Cloud의 Python version이 **3.11**인지 |
| "스프레드시트를 찾을 수 없습니다" | ① Google **Drive API** 사용 설정 여부 (2-3단계)<br>② `spreadsheet_name`이 실제 시트 이름과 같은지<br>③ 서비스 계정이 **편집자**로 공유되어 있는지 (2-5단계) |
| 인증 오류가 난다 | Secrets의 `private_key` 값에 `\n`이 그대로 남아 있는지 |
| 모둠 메뉴가 안 보인다 | `모둠` 탭이 있는지, 헤더 순서가 맞는지 (6단계) |
| AI 피드백이 안 나온다 | 한 종목에 기록이 **2회 이상** 쌓였는지 / `GEMINI_API_KEY` 입력 여부 |

문제가 계속되면 이 저장소의 **Issues** 탭에 남겨 주세요.

## 🔒 개인정보 보호

- 학생 비밀번호는 평문이 아닌 **해시(SHA-256)** 형태로 저장됩니다.
- 초기 비밀번호는 전원 `0000`으로 동일합니다. 데모 체험과 최초 로그인 안내를 쉽게 하기 위한 설정이므로, **실제 학급 운영을 시작할 때는 학생별로 다른 비밀번호로 변경**하시기를 권합니다. 학생 관리 메뉴에서 바꿀 수 있습니다.
- 학생은 자신의 계정으로 로그인한 경우에만 자신의 기록을 열람할 수 있습니다.
- 데이터는 담당 교사가 소유한 구글 시트에 저장되며, 접근 권한은 교사에게만 있습니다.
- 서비스 계정 JSON 키는 **절대 저장소에 올리지 마세요.** 반드시 Streamlit Secrets에만 입력합니다.

> **복제해 사용하시는 경우**, `app.py`의 `PRIVACY_POLICY_MD` 항목에 소속 학교와 담당자 연락처를 기재하여 운영하시기 바랍니다. 건강 정보는 민감정보에 해당하므로, 학기 초 학부모 안내와 동의 절차를 함께 진행하시길 권합니다.
